# 把 RoPE 沿 (frame, height, width) 三路拆分 / Splitting RoPE three ways: frame, height, width

> **一句话 / In one line**: Wan2.1 给 head_dim 切三段,前 `c - 2*(c//3)` 维编时间轴 t、中间 `c//3` 维编 h、最后 `c//3` 维编 w,三段单独算复数 RoPE 再拼起来,等价于把 1D RoPE 同时挂到视频的三个坐标轴上。 / Wan2.1 splits each head dim into three slices — temporal, height, width — applies 1-D RoPE independently to each slice in the complex domain, then concatenates. Effectively a 1-D RoPE replicated along all three video coordinates.

## 为什么重要 / Why this matters

DiT 处理视频时,patchify 后每个 token 同时带有三个坐标 `(f, h, w)`。最朴素的位置编码是给每个 `(f, h, w)` 整体一个 learnable 向量,参数量爆炸还不能外推到更长视频。Wan2.1(以及 Sora、CogVideoX、lingbot-va)都采用 **3D 因式分解 RoPE**:把 head_dim 切成三段,每段单独编一个坐标轴的 RoPE,然后通过 `complex multiply` 旋转 query/key。这个做法的好处是 (1) 无参数 — RoPE 是函数式的,(2) 训练时见过 16 帧,推理时可以拉到 100+ 帧依然正常,(3) 三个轴各自独立,改 height 不影响 frame 的编码。今天这 40 行就是它的工业级实现。

When a DiT processes video, every patchified token carries three coordinates `(f, h, w)`. The naive choice — learnable embeddings indexed by `(f, h, w)` — blows up parameter count and prevents extrapolation to longer clips. Wan2.1 (and Sora, CogVideoX, lingbot-va) all use **3-D factorized RoPE**: slice the head dim into three groups, run 1-D RoPE on each axis independently, multiply into Q/K as complex rotations, concatenate. The wins are (1) parameter-free, (2) trained on 16 frames generalises to 100+ frames cleanly, (3) the three axes are independent so changing resolution doesn't disturb temporal encoding. These 40 lines are the production implementation.

## 代码 / The code

`Wan-Video/Wan2.1` — [`wan/modules/model.py`](https://github.com/Wan-Video/Wan2.1/blob/9737cba9c1c3c4d04b33fcad41c111989865d315/wan/modules/model.py#L32-L70)

```python
@amp.autocast(enabled=False)
def rope_params(max_seq_len, dim, theta=10000):
    assert dim % 2 == 0
    freqs = torch.outer(
        torch.arange(max_seq_len),
        1.0 / torch.pow(theta,
                        torch.arange(0, dim, 2).to(torch.float64).div(dim)))
    freqs = torch.polar(torch.ones_like(freqs), freqs)
    return freqs


@amp.autocast(enabled=False)
def rope_apply(x, grid_sizes, freqs):
    n, c = x.size(2), x.size(3) // 2

    # split freqs
    freqs = freqs.split([c - 2 * (c // 3), c // 3, c // 3], dim=1)

    # loop over samples
    output = []
    for i, (f, h, w) in enumerate(grid_sizes.tolist()):
        seq_len = f * h * w

        # precompute multipliers
        x_i = torch.view_as_complex(x[i, :seq_len].to(torch.float64).reshape(
            seq_len, n, -1, 2))
        freqs_i = torch.cat([
            freqs[0][:f].view(f, 1, 1, -1).expand(f, h, w, -1),
            freqs[1][:h].view(1, h, 1, -1).expand(f, h, w, -1),
            freqs[2][:w].view(1, 1, w, -1).expand(f, h, w, -1)
        ],
                            dim=-1).reshape(seq_len, 1, -1)

        # apply rotary embedding
        x_i = torch.view_as_real(x_i * freqs_i).flatten(2)
        x_i = torch.cat([x_i, x[i, seq_len:]])

        # append to collection
        output.append(x_i)
    return torch.stack(output).float()
```

## 逐行讲解 / What's happening

1. **`rope_params` 算频率表 / Frequency table**:
   - 中文:`torch.outer(position, 1/theta^(2i/d))` 给每个 (位置, 维度对) 算一个角度 `θ_pos,i`;`torch.polar(ones, θ)` 把它变成模长为 1 的复数 `cos θ + i·sin θ`。这就是 1D RoPE 在频域的形式。
   - English: `torch.outer(position, 1 / theta**(2i/d))` produces angle `θ_{pos,i}` for every (position, dim-pair); `torch.polar(ones, θ)` converts each angle into the unit-modulus complex number `cos θ + i·sin θ`. That's standard 1-D RoPE in the frequency domain.

2. **三段切分:`c - 2*(c//3), c//3, c//3` / Three-way split**:
   - 中文:`c = head_dim // 2`(每两维一对实数 → 一个复数)。三段相加正好等于 `c`,而且 t 段最大(收吸收余数),保证 `head_dim` 不被偶数三分整除时也能 work。
   - English: `c = head_dim // 2` because every adjacent pair of reals becomes one complex number. The three slice sizes sum exactly to `c`, with the temporal slice absorbing the remainder so any head_dim works even when not divisible by 6.

3. **`grid_sizes` 是 per-sample 的 (f, h, w) / Per-sample shapes**:
   - 中文:batch 里不同视频可能 `(f, h, w)` 不同(比如做 multi-resolution 训练),所以 `grid_sizes` 是 `[B, 3]` 而不是全局常量。`for i in batch:` 这条 Python 循环就是处理 batch 不齐的情形,代价是没法用 vmap 但工程上够用。
   - English: different samples in the batch may have different `(f, h, w)` (multi-resolution training), so `grid_sizes` is `[B, 3]`. The Python `for i` loop walks the batch — not vmap-friendly but fine in practice.

4. **`torch.view_as_complex(...reshape(...,-1,2))` / Reinterpret reals as complex**:
   - 中文:把 `head_dim` 维度重排成 `[head_dim/2, 2]`,然后 `view_as_complex` 把最后那一对当成 `(real, imag)`,得到 complex tensor。没有 memory copy。
   - English: reshape head_dim into pairs `[head_dim/2, 2]` and reinterpret each pair as `(real, imag)`. No data is copied — it's just a stride view.

5. **三个 broadcast 拼接 / Three broadcasts concatenated**:
   - 中文:`freqs[0][:f].view(f, 1, 1, -1).expand(f, h, w, -1)` 把时间频率沿 h、w 重复;另外两条同理。三段拼起来后,每个 `(f_idx, h_idx, w_idx, dim_idx)` 位置的复数旋转角度精确等于 `θ_t(f_idx) ⊕ θ_h(h_idx) ⊕ θ_w(w_idx)`。
   - English: `freqs[0][:f].view(f,1,1,-1).expand(f,h,w,-1)` broadcasts the temporal frequencies along the spatial axes; the other two do the same for h and w. After concat, the complex rotation at `(f_idx, h_idx, w_idx, dim_idx)` is the per-axis frequency for the slice that dim belongs to — exactly factorized RoPE.

6. **`x_i * freqs_i` 是复数乘 = 实数空间的 2D 旋转 / Complex multiply ≡ 2-D rotation**:
   - 中文:复数 `(a + bi) * (cos θ + i sin θ)` 转回实数对就是 `(a·cos θ - b·sin θ, a·sin θ + b·cos θ)`,即 2D 旋转矩阵 `R(θ)` 作用在 `(a, b)` 上。这就是 RoPE 的核心数学:把每一对 (real, imag) 当成 2D 平面上的向量,绕原点旋转 θ。
   - English: `(a + bi) * (cos θ + i sin θ)` equals `(a cos θ - b sin θ, a sin θ + b cos θ)`, i.e. the 2-D rotation matrix `R(θ)` applied to `(a, b)`. That's the entire RoPE math — every consecutive pair of dims gets rotated by the per-position angle.

7. **`x_i[seq_len:]` 透传 / Pass-through for non-video tokens**:
   - 中文:序列后面可能挂了 text token 或 register token,它们不需要 3D RoPE,直接 `torch.cat([x_i, x[i, seq_len:]])` 透传过去。这是给"文本条件 + 视频 latent 共享一个序列"留的口子。
   - English: any tokens past `seq_len` (text tokens, register tokens) bypass the 3-D RoPE and are concatenated unchanged. That's the hook for mixing text-conditioning tokens with video latent tokens inside one attention stream.

## 类比 / The analogy

像给"国际航班"贴三层标签:一段编国家(t),一段编城市(h),一段编街道(w)。三段标签互不重叠,所以改街道不影响国家和城市。每个标签都是个"角度"(钟表指针),让两条相同 token 通过角度差就能反推出它们的相对距离。整体效果是:不需要记住"具体经纬度",只要知道"两个 token 之间相差的角度",注意力就能算对相对位置。

Think of triple-labelling international flights: one slice encodes country, one city, one street. The labels never overlap, so changing the street doesn't disturb the country tag. Each tag is an angle (clock hand); two tokens carrying the same label can read their *difference* and recover relative distance — no absolute coordinate ever needed. That's RoPE's whole appeal: attention works on relative distance, not absolute index.

## 在 nanoVLA / nanoWAM 中的位置 / Where this lives in your nano-WAM

中文:在 nanoWAM 里这就是 `nano/wam/blocks/rope_3d.py`,跟 `patchify.py`(把 VAE 输出的 `[B, C_lat, T, H, W]` reshape 成 `[B, seq_len, dim]`)是同级。上游是 VAE 的输出 + Patchify 把 (t, h, w) 算成 grid_sizes;下游是 attention 层,query 和 key 各自先调 `rope_apply` 再喂 flash-attention。如果你省掉这一步,DiT 就完全无法分辨"这个 token 在第 5 帧还是第 50 帧",视频会失去时间一致性 —— 训练时 loss 还能降,但生成出来的视频会像幻灯片乱序。生产实现还要加:**dynamic shape 缓存**(对同一个 `(f, h, w)` 缓存 `freqs_i`,batch 多了就直接复用)、**长视频外推**(把 theta 调大或者用 NTK-aware scaling),以及 **与 cross-attention 文本分离**(文本用普通 1-D RoPE 或 learnable embedding,不和视频共用)。

English: in nanoWAM this is `nano/wam/blocks/rope_3d.py`, sitting alongside `patchify.py` that reshapes `[B, C_lat, T, H, W]` into `[B, seq_len, dim]`. Upstream is the patchify step that produces `grid_sizes`; downstream is the attention layer, where Q and K each call `rope_apply` before flash-attention. Skipping this leaves the DiT unable to tell frame 5 from frame 50 — training loss still drops, but generated clips look like shuffled slides. Production additions: **shape cache** for `freqs_i` keyed by `(f, h, w)`, **long-video extrapolation** via larger theta or NTK-aware scaling, and **separate positional handling for text tokens** (give cross-attention text its own 1-D RoPE rather than reuse the video 3-D split).

## 自己跑一遍 / Try it yourself

```python
# pip install torch
import torch

def rope_params(max_seq_len, dim, theta=10000):
    assert dim % 2 == 0
    freqs = torch.outer(
        torch.arange(max_seq_len, dtype=torch.float64),
        1.0 / theta ** (torch.arange(0, dim, 2, dtype=torch.float64) / dim))
    return torch.polar(torch.ones_like(freqs), freqs)

def rope_apply_3d(x, f, h, w, freqs_t, freqs_h, freqs_w):
    # x: [seq_len, n_heads, head_dim], seq_len = f*h*w
    seq, nh, hd = x.shape
    c = hd // 2
    cf, ch, cw = c - 2 * (c // 3), c // 3, c // 3
    x_c = torch.view_as_complex(x.to(torch.float64).reshape(seq, nh, -1, 2))
    ft = freqs_t[:f, :cf].view(f, 1, 1, -1).expand(f, h, w, -1)
    fh = freqs_h[:h, :ch].view(1, h, 1, -1).expand(f, h, w, -1)
    fw = freqs_w[:w, :cw].view(1, 1, w, -1).expand(f, h, w, -1)
    freqs = torch.cat([ft, fh, fw], dim=-1).reshape(seq, 1, -1)
    return torch.view_as_real(x_c * freqs).flatten(2).float()

f, h, w, nh, hd = 4, 2, 2, 2, 12        # tiny shapes
seq = f * h * w
x = torch.randn(seq, nh, hd)
ft = rope_params(f, (hd // 2 - 2 * (hd // 6)) * 2)
fh = rope_params(h, (hd // 6) * 2)
fw = rope_params(w, (hd // 6) * 2)
y = rope_apply_3d(x, f, h, w, ft, fh, fw)
print("shape preserved:", y.shape, x.shape)
print("magnitudes preserved:", torch.allclose(x.norm(dim=-1), y.norm(dim=-1), atol=1e-4))
```

## 注意事项 / Caveats / when it breaks

- **`head_dim` 必须能被 6 近似整除** / **`head_dim` should be roughly divisible by 6**: 中文 — 三段拆分时 t 段吃余数,但如果 head_dim=64 → c=32 → 三段 12, 10, 10,频率分辨率会偏差。SOTA 配置一般用 head_dim=128 (-> 22, 22, 20)。 / English — split with `c = head_dim // 2`. Best-behaved when `head_dim` is divisible by 6; otherwise the temporal slice absorbs the slack and you get slightly asymmetric frequency resolution.
- **theta 决定外推能力** / **`theta` controls extrapolation**: 中文 — Wan2.1 默认 theta=10000,适合训练分辨率;想生成更长视频可以放大 theta 或用 dynamic NTK scaling。 / English — `theta=10000` matches the training resolution. Longer clips need either a larger theta or dynamic NTK-aware scaling, otherwise extrapolation degenerates.
- **fp64 不能掉** / **Don't drop fp64**: 中文 — `@amp.autocast(enabled=False)` + `.to(torch.float64)` 是有意的,RoPE 频率对精度极敏感,fp16 会把高频角度全砸成 0。 / English — the `@amp.autocast(enabled=False)` and `.to(torch.float64)` are not optional. RoPE frequencies need fp64; fp16 collapses the high-frequency angles to zero.
